## Quick Start

### Using Docker (Recommended)

```bash
# Build computational environment
docker build -t {{PROJECT_NAME}}-env .

# Run analysis in container
docker run --rm -v $(pwd):/project -w /project {{PROJECT_NAME}}-env \
  Rscript -e 'rmarkdown::render("analysis/paper/paper.Rmd")'

# Start RStudio Server (http://localhost:8787, user: rstudio, pass: rstudio)
docker run -p 8787:8787 -e PASSWORD=rstudio {{PROJECT_NAME}}-env
```

### Local Development

```bash
# Restore R package environment
Rscript -e 'renv::restore()'

# Render manuscript
Rscript -e 'rmarkdown::render("analysis/paper/paper.Rmd")'

# Run tests
Rscript -e 'devtools::test()'
```

## Project Structure

```
.
├── analysis/
│   ├── data/

│   │   └── references.bib    # Bibliography
│   ├── figures/              # Generated plots and visualizations

├── R/                        # Reusable functions
├── man/                      # Function documentation
├── tests/                    # Unit tests
├── Dockerfile                # Computational environment specification
├── renv.lock                 # R package versions
└── DESCRIPTION               # Project metadata
```

### File Organization Principles

Following Marwick et al. (2018), files are organized into three categories:

- **Read-only**: Raw data, metadata (never modified after collection)
- **Human-generated**: Code, documentation, manuscripts
- **Project-generated**: Processed data, figures, analysis outputs
